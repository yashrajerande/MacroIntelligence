# Executive Summary Style Guide

## The Standard
Write as if Neelkanth Mishra is checking your India analysis, Charlie Munger is checking your logic, and the Economist's editor is checking your prose. If any of them would object, rewrite.

## Paragraph Labels
  01 — India Macro Regime
  02 — Global Macro Regime
  03 — Liquidity Conditions
  04 — Equity + Real Estate Implications
  05 — Key Risks to Monitor

## Mandatory Rules
- Open each paragraph with the single most important number.
- Use em-dashes for causation chains: "Brent $95 — CAD widens $15bn — INR at 87 — imported inflation +40bps".
- Use arrows for direction: "→ rate cut → mortgage repricing → housing demand".
- Never start a sentence with "The".
- No paragraph longer than 4 sentences.
- Wrap all key figures in <strong> tags.
- Final sentence of para 05: the single most important data point this week.
- Every sentence must contain at least one number or a specific falsifiable claim.

## The Mishra Test
Before writing about India GDP, ask: "What does this number actually measure? What's happening in the informal economy that this misses? What do high-frequency proxies (GST, e-way bills, UPI, toll collections) say?"

## The Munger Test
Before any bullish statement, invert: "What would make this wrong?" State the inversion explicitly. "GDP at 7.8% looks robust — unless you note that GVA was 6.1% and the gap is statistical discrepancy, not real output."

## The Economist Test
Read each sentence aloud. If it sounds like a press release, a government communiqué, or a sell-side initiation report, rewrite it. It should sound like an editorial written by someone with nothing to sell and no career risk.

## Anti-Patterns (delete on sight)
- Describing what everyone already knows ("CPI came in at 3.2%") without saying what it MEANS
- Listing numbers without connecting them causally
- Using the word "despite" to hand-wave away contradictions instead of explaining them
- Any sentence that works equally well for any country in any quarter
- Hedging with "could" or "may" when the data supports a directional call
