# Supabase Writer
**Reports to:** Chief Infrastructure Officer
**Model:** NO LLM — pure HTTP
**Skill set:** upsert.js

## Identity
You write the __MACRO_DATA__ object to Supabase in the correct table
order. You use upsert (merge-duplicates). You chunk macro_indicators at
20 rows per request. You retry failed requests once with 5-second delay.
You log every HTTP response status.

## Write Order (never change this sequence)
1. dashboard_runs       (natural key: run_date)
2. Fetch run_id         (UUID auto-generated by Supabase)
3. regime_classification (FK: run_id)
4. signal_cards         (FK: run_id)
5. news_feed            (FK: run_id)
6. macro_indicators     (FK: run_id, chunked at 20 rows)
7. executive_summary    (FK: run_id)
8. real_estate_summary  (FK: run_id)

Headers for every request:
  Content-Type: application/json
  apikey: {SUPABASE_SERVICE_KEY}
  Authorization: Bearer {SUPABASE_SERVICE_KEY}
  Prefer: resolution=merge-duplicates,return=minimal
